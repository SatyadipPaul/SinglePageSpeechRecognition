Visual Architecture: The "Layered Orb" ConceptThe organic, morphing holographic effect is NOT achieved through a complex custom WebGL shader. Instead, it utilizes CPU-side vertex deformation applied to multiple overlapping, high-resolution sphere geometries utilizing specific blending modes to create an illusion of volumetric, glowing plasma.2.1 Geometry SpecificationsShape: THREE.SphereGeometryResolution: 64 segments by 64 rings. Critical: High resolution is required for smooth sine-wave deformation without jagged edges.State Caching: Upon creation, the original untouched vertex positions MUST be cached.geometry.userData.originalPositions = geometry.attributes.position.clone();(Failure to cache and reference original positions during the render loop will result in exponential deformation and mesh explosion).2.2 Material & Layer SpecificationsFour distinct spheres are created and rendered in the exact order below.Base Material: THREE.MeshBasicMaterial (Unaffected by scene lighting to maintain pure color values).Shared Properties: transparent: true, depthWrite: false, wireframe: false.Layer IndexRoleBase RadiusColor (Hex)OpacityBlending Mode0Outer Aura (Cyan)1.60x00ffff0.4THREE.AdditiveBlending1Mid Glow (Magenta)1.50xff00ff0.4THREE.AdditiveBlending2Inner Core (Blue)1.40x0044ff0.6THREE.AdditiveBlending3Solid Center (White)1.10xffffff0.8THREE.NormalBlendingNote: Additive blending on the outer layers causes the colors to mix into bright white/neon hues where they intersect.3. The Animation Loop & MathematicsThe animation relies on modifying the distance of each vertex from the sphere's center based on 3D sine waves and time.3.1 Global Variablestime: Calculated as Date.now() * 0.001 per frame.smoothedAudioLevel: See section 4.index: The array index of the current layer being processed (0 to 3).3.2 Layer RotationEach layer rotates independently on the Y and Z axes to create a tangled, chaotic feel.rotation.y += 0.002 * (index + 1)rotation.z += 0.001 * (index + 1)3.3 Vertex Deformation Algorithm (Pseudo-shader in JS)For every vertex in posAttr.count, the following logic is applied:Retrieve Original Position: v.fromBufferAttribute(origPos, i);Calculate Unique Frequencies per Layer: To ensure layers weave through each other, frequency multipliers vary by index.freqX = 1.2 + index * 0.3freqY = 1.5 + index * 0.2freqZ = 1.1 + index * 0.4Calculate Time Offset: timeMult = time * (1 + index * 0.5)The Noise Function: A combination of sine and cosine waves across all three axes.noise = Math.sin(v.x * freqX + timeMult) * Math.cos(v.y * freqY + timeMult) * Math.sin(v.z * freqZ + timeMult);
Apply Amplitude: * amp is determined by audio input (see Section 4).newRadius = baseRadius + (noise * amp)Update Vertex:v.normalize().multiplyScalar(newRadius)posAttr.setXYZ(i, v.x, v.y, v.z)Flag Update: posAttr.needsUpdate = true4. Audio Reactivity PipelineThe visual intensity is directly driven by the microphone via the Web Audio API.Audio Context Setup: AnalyserNode created with fftSize = 256.Data Extraction: In the render loop, fetch getByteFrequencyData().Calculate Average Volume: Sum all frequencies and divide by dataArray.length.Normalization: currentAudioLevel = averageVolume / 50; (Maps normal speaking volume to roughly 1.0).Smoothing (Crucial for fluid visuals): smoothedAudioLevel += (currentAudioLevel - smoothedAudioLevel) * 0.15;Amplitude Mapping:Idle Amplitude (baseAmp): 0.2 (if mic is active) or 0.05 (if mic is off).Final Deformation Amplitude (amp): baseAmp + (smoothedAudioLevel * 0.8).Global Scale Pulsation: In addition to vertex deformation, the entire layer scales up based on audio:scale = 1 + (smoothedAudioLevel * 0.2); layer.scale.set(scale, scale, scale);5.
